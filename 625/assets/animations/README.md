This is a deterministic, exactly solved finite illustration for the repository
landing pages. Seed 78 was deliberately selected to make the gap and both
partitions visually instructive. It is not a representative sample, statistical
evidence, or evidence for the asymptotic theorem, and it is not used in the
proof.

## Certified instance

- Model: `G(12,1/2)` generated by `splitmix64-v1/high-bit/p=1/2` with seed
  `78`.
- Graph SHA-256: `58dc4b710fefe0fabcd7d0b855475d2d95f8769dd3304c29e1686c81c3ccf81b`.
- Exact invariants: `alpha=5`, `omega=6`, `chi=6`, `zeta=3`; hence
  `chi-zeta=3`.
- The six displayed color classes are independent sets.
- The three displayed cochromatic parts are two cliques and one independent
  set.
- A displayed `K6` certifies `chi>=6` and excludes a partition into two
  independent sets. An independent triple excludes a partition into two
  cliques. An induced `2K2` excludes a split into one clique and one
  independent set. Together with the displayed witnesses, these certify the
  exact values.

The full edge list, witnesses, render settings, hashes, and output sizes are
machine-readable in
[`erdos625-coloring-example.json`](erdos625-coloring-example.json).

## Rebuild

From the repository root, use Python 3.10 or newer with Pillow installed and
either FFmpeg on `PATH` or the `imageio-ffmpeg` package available:

```powershell
python 625/experiments/render_erdos625_animation.py
```

If FFmpeg is not on `PATH`, pass it explicitly:

```powershell
python 625/experiments/render_erdos625_animation.py --ffmpeg C:\path\to\ffmpeg.exe
```

The script imports the repository's exact solver, reconstructs and validates
the fixed graph and all witnesses, then renders both formats. It uses no
random layout state and records both the generator-script hash and imported
exact-solver hash in the JSON sidecar. The sidecar stores the FFmpeg executable
basename, discovery source, version, Python and Pillow versions, and font
basenames, not machine-specific paths. Graph generation and layout are
deterministic; byte-identical media also depend on the recorded fonts, Pillow,
and FFmpeg implementation.
